# 04 — 文件格式杂谈与学习边界（PDF、二进制、《龙书》）

## 文本文件 vs 二进制文件（实用区分）

| | 文本文件 | 二进制文件 |
|---|----------|------------|
| **常见例子** | `.txt`、`.toml`、`.md`、多数源码 | `.pdf`、`.png`、`.exe`、压缩包 |
| **约定** | 主要用 **UTF-8**（或 ASCII）编码，编辑器可读 | 结构由格式规范定义，**不保证**整文件当人类可读文本 |
| **本质** | 磁盘上都是**字节**；是否「像文本」取决于编码与用途 |

`uv.lock`、`.toml` 一般属于**文本**，可直接在 Cursor 里阅读。

## PDF 是什么？为什么编辑器里常「不好直接看」？

- **PDF** 是复杂的**文档格式规范**（对象、流、压缩、字体子集、矢量指令等），**不是**一段 UTF-8 小说。
- **能看 PDF** = 需要 **解析格式 + 排版渲染**。
- **Cursor / VS Code** 核心是**代码编辑**；是否内建 **PDF 预览**取决于是否安装扩展、产品策略。打不开时可用**系统阅读器、浏览器**或安装 **PDF 预览扩展**。

这与「PDF 存的是什么」无关：PDF 仍然是文件，只是**格式专用于版式文档**，不是「编辑器默认识别」的那类纯文本。

## 《编译原理》（龙书）会讲这些吗？

通常说的 **《龙书》**（*Compilers: Principles, Techniques, and Tools*）主线是：**词法/语法分析、语义、优化、代码生成** 等**编译器**理论。

- **不会**专门讲 `uv.lock`、包管理器设计细节。
- **不会**用课程主线讲 **PDF 文件格式** 或 **HTTP / TCP** 工程细节。

若关心「各种文件格式如何组织字节」，应查阅**各格式的官方规范**或专门书籍；若关心网络，看**计算机网络**教材；若关心包管理，看 **uv / pip** 文档与 **PEP**。

## 和本篇相关的误区澄清

- **`sync`（uv）** ≠ TCP 的 **SYN**（拼写相似，领域不同）。
- **元数据** ≠ 「神秘数据」，就是**描述项目/资源的信息**。
- **学习顺序**：先建立 **工具链与依赖**（见前两篇笔记），再深入 **Gateway / LangGraph**（第三篇），格式杂谈本篇作补充即可。
