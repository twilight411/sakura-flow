# 02 — 依赖锁定与项目文件（TOML、Lock、Sync、元数据）

## TOML 是什么？

- **TOML** 是一种**人类可读的配置文件格式**（分段、键值对），常用于 `pyproject.toml`、部分 lock 文件等。
- **`pyproject.toml`**：Python 项目的**标准清单**，说明包名、版本、依赖范围、构建方式等。工具（uv / pip / hatch）约定读这个文件。

## 元数据（metadata）

- **定义**：**关于数据的数据**——描述「这个项目/包是谁、什么版本、依赖谁」，**不是**业务里用户聊天内容那种核心数据。
- **例子**：`pyproject.toml` 里的 `name`、`version`、`requires-python`、作者信息，都属于**项目元数据**。

## `uv.lock` 是什么？为什么要锁？

- **作用**：把依赖解析结果**钉死到具体版本**（以及哈希等信息），实现**可重复安装**。
- **解决的问题**：
  - 仅写 `fastapi>=0.115` 时，不同时间、不同机器可能解析出**不同**的满足条件的版本。
  - 锁文件让「你、同事、CI」在 `uv sync` 后**尽量装同一套版本**，减少「我这能跑你那报错」。
- **和「本机已经装过」的关系**：**以 lock + `pyproject.toml` 为权威**。若 `.venv` 里版本与 lock 不一致，`uv sync` 会**调整**到 lock 描述的状态，而不是「本机有啥就用啥」。

## `uv sync` 里的 sync 是什么意思？

- 这里是 **synchronize（同步、对齐）**：让 **磁盘上的虚拟环境（`.venv`）** 与 **`pyproject.toml` / `uv.lock`** 的描述**一致**。
- **与 TCP 无关**：网络里的 **SYN / SYN-ACK / ACK** 是传输层握手缩写；**不要**和 `uv sync` 混为一谈。

## `uv.lock` 是文本还是二进制？

- **通常是纯文本**（常见为 TOML 风格），UTF-8 编码，用编辑器可直接阅读。
- **二进制文件**一般指：不假定整文件为可读文本（如 PDF、图片、部分压缩包），用记事本打开会乱码或大量不可读内容。

## 为什么一行命令会「冒出很多文件」？

分两类理解：

### `uv init`

- 不是只改一个配置，而是**脚手架生成**：一次性创建 **可被构建工具识别** 的目录结构，例如：
  - `pyproject.toml`、`.python-version`、`README.md`
  - `src/<包名>/` 等源码树（带 `--package` 时）

### `uv sync`

- 会创建或更新 **虚拟环境目录 `.venv/`**（解释器、 `site-packages` 等），体积可能很大。
- 可能生成或更新 **`uv.lock`**（锁定解析结果）。

**原理**：Python 依赖是**许多包 + 确定版本**的组合；工具必须把包装进 venv，所以会看到大量文件。

## 小结表

| 文件/目录 | 角色 |
|-----------|------|
| `pyproject.toml` | 声明项目信息与依赖**范围** |
| `uv.lock` | 锁定依赖**精确版本**，利于复现 |
| `.venv/` | 本机实际安装的解释器与包（一般不入库，靠 lock 复现） |
