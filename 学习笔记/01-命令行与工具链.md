# 01 — 命令行与工具链（Windows / uv）

## PowerShell 里没有 `touch`

- **原因**：`touch` 常见于 Linux/macOS；**Windows PowerShell 默认没有**同名命令。
- **等价操作**：
  - **新建空文件**：`New-Item -Path .\文件名.txt -ItemType File`
  - **只更新已有文件的修改时间**：`(Get-Item .\文件名).LastWriteTime = Get-Date`
- **可选**：安装 **Git for Windows** 后，在 **Git Bash** 里仍可使用 `touch`。

## `uv` 提示「无法识别」

- **含义**：当前终端在 **`PATH` 环境变量** 里找不到名为 `uv` 的可执行文件。
- **常见安装位置**（官方安装脚本）：`%USERPROFILE%\.local\bin\uv.exe`
- **处理**：把 **`C:\Users\<用户名>\.local\bin`** 加入**用户级** `Path`，**关闭并重新打开终端**（必要时重启 IDE），使新 `PATH` 生效。
- **验证**：`uv --version`

## 为什么「新开窗口」有时仍找不到命令？

- 若 **`PATH` 从未写入用户/系统环境变量**，仅当前会话里临时改过，新窗口会丢。
- 若 **IDE 缓存旧环境**，需**重启 Cursor / VS Code** 再试。
- **临时绕过**：用完整路径执行，例如 `& "$env:USERPROFILE\.local\bin\uv.exe" --version`

## `uv init` 建议写清路径

教程里若只写 `uv init --name sakura-flow --package`，容易漏掉「在哪个目录初始化」。更稳妥：

```powershell
cd D:\flows\sakura-flow\backend
uv init . --name sakura-flow --package -p 3.12
```

- **`.`**：当前目录（`backend`）作为项目根。
- **`-p 3.12`**：绑定 Python 3.12，并辅助生成 `.python-version`。

具体选项以本机 `uv init --help` 为准。

## 与本仓库相关的命令备忘

| 命令 | 作用（一句话） |
|------|----------------|
| `uv init` | 在指定路径生成 Python 项目骨架（`pyproject.toml` 等） |
| `uv sync` | 按清单创建/更新 `.venv` 并安装依赖，使环境与声明一致 |
| `uv run <命令>` | 在项目的虚拟环境里执行命令 |
