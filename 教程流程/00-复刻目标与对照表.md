# 00 — 复刻目标与对照表

## DeerFlow 2.0 在解决什么问题（你复刻时要对齐的「大方向」）

官方定位（摘自上游 README）：**Super Agent Harness**——编排 **子代理（sub-agents）**、**记忆（memory）**、**沙箱（sandbox）**，并通过可扩展的 **Skills / Tools** 扩展能力。实现上典型地分为：

1. **图与运行时**：LangGraph 定义的 agent 图 + checkpoint + SDK。
2. **网关层**：FastAPI（或等价）对外提供 HTTP / SSE，与前端、线程、运行状态对接。
3. **前端**：Next.js 对话与流程 UI，通过 `@langchain/langgraph-sdk` 等与后端协作。
4. **配置与运维**：根目录 `config.yaml`、`.env`、`Makefile`、可选 Docker + Nginx 反代。

你在 `sakura-flow` 里复刻时，不必逐文件拷贝，但**这四块都要有清晰的落点**。

## 上游仓库地图（对照用）

以 `deer-flow` 根目录为例（随版本可能微调）：

| 路径 | 作用 |
|------|------|
| `Makefile` | 统一入口：`config`、`install`、`dev`、`docker-*` |
| `config.example.yaml` / `config.yaml` | 模型与全局行为 |
| `scripts/` | `serve.sh`、`configure.py` 等启动与配置脚本 |
| `backend/` | Python：Gateway（`app/gateway`）、`langgraph.json`、workspace 包 `packages/harness` |
| `frontend/` | Next.js：pnpm、`pnpm run dev` |
| `docker/` | Nginx 配置、容器编排等 |
| `skills/` | 可装载技能示例或元数据 |

## 命名与边界建议

- **项目名**：仓库、Python `project.name`、前端 `package.json` 的 `name` 统一成 `sakura-flow` 或你喜欢的标识（避免与上游 `deer-flow` 混淆即可）。
- **Python 包名**：上游用 `deerflow`（harness 内）；你可使用 `sakura` / `sakura_flow` 等，但保持 **一处改名、全局一致**（`langgraph.json` 里 `graphs` 的入口也要指向你的模块）。
- **学习边界**：上游代码仅作**结构与 API 用法**参考；业务逻辑、密钥、许可证请遵循上游与原项目要求。

## 复刻完成的「验收」参考（自拟）

按阶段勾选（不必一次做完）：

- [ ] 根目录能 `make install`（或等价）装好前后端依赖。
- [ ] 能 `make config` 生成本地配置，并理解 `models` 最少配置。
- [ ] 后端能单独启 Graph dev + Gateway（或与上游同等拓扑）。
- [ ] 前端能连上本地网关 / LangGraph 端点，完成一次对话或最小 smoke。
- [ ]（可选）Docker 路径能起服；Sandbox 能按你环境拉到镜像或降级为本地模式。
