# 01 — 脚手架与仓库骨架（第一步）

本章目标：在 **`D:\flows\sakura-flow`** 里搭出与 DeerFlow **同形态**的空仓库——目录齐全、能 `git`、有占位配置，但不必立即实现业务代码。

## 1. 前置：工具装齐（只做一次）

在动手前确认本机有：

- **Git**
- **Node.js 22+**（`node -v`）
- **pnpm**（`pnpm -v`；若无：`npm install -g pnpm`）
- **Python 3.12+**（`python --version` 或 `py -3.12 --version`）
- **uv**（建议安装，见 [uv 文档](https://docs.astral.sh/uv/)）

> Windows 上后续 `Makefile` 可能依赖 Git Bash 跑 `serve.sh`；若你在 Win 上遇到脚本问题，上游已提供 `scripts\run-with-git-bash.cmd` 一类兼容，复刻时也可先 WSL2 或纯手敲 PowerShell 等价流程。

## 2. 创建空目录并初始化 Git（请尽量手敲命令）

在 PowerShell 或终端里**自己键入**（路径按你本机调整）：

```powershell
cd D:\flows
mkdir sakura-flow
cd sakura-flow
git init
```

建议首提交前加 `.gitignore`（可先放最小集合，后续再补）：

```gitignore
# Python
.venv/
__pycache__/
*.py[cod]
.pytest_cache/
.ruff_cache/

# Node
node_modules/
.next/
dist/

# Env & local config（若你希望我）
.env
.env.local
config.yaml

# IDE
.idea/
.vscode/
```

## 3. 顶层目录结构（对齐 DeerFlow）

在 `sakura-flow` 根目录下创建下列**空文件夹**（名称与上游一致，便于对照）：

```text
sakura-flow/
  backend/           # Python + uv + LangGraph
  frontend/          # Next.js + pnpm
  scripts/           # serve、configure 等
  docker/            # nginx、compose 等（后期）
  skills/            # 技能元数据或示例（后期）
  docs/              # 可选：你自己的设计笔记
  教程流程/          # 本教程（可保留）
```

PowerShell 示例（可手敲）：

```powershell
mkdir backend, frontend, scripts, docker, skills, docs
```

## 4. 根级占位文件

### 4.1 `config.example.yaml`

先复制 DeerFlow 的**结构意图**即可，不必和上游一模一样。最小可写：

```yaml
# 示例：复制为 config.yaml 后填写；真实字段对齐 ../deer-flow/config.example.yaml
models: []
```

复刻中后期把上游 `deer-flow/config.example.yaml` 里的段落**按需**迁移进来。

### 4.2 `Makefile`（可先 stub）

目标：后续能扩展为 `make config`、`make install`、`make dev`。第一课可以只写：

```makefile
.PHONY: help
help:
	@echo "sakura-flow — run: make help (后续补充 config/install/dev)"
```

等你接上 `scripts/` 后再把 `deer-flow/Makefile` 里的目标**一条条搬**并改成你的路径。

### 4.3 `README.md`（项目自述）

写两三句话：项目目标（复刻 DeerFlow 架构）、技术栈、如何进入 `教程流程/`。无需照抄上游长 README。
